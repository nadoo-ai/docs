## Setting Up the Documentation

### Clone the Repository

```bash
git clone https://github.com/nadoo-ai/docs.git
cd docs
```

### Install Dependencies

```bash
npm install
```

### Run the Development Server

Start the Mintlify development server:

```bash
npm run dev
```

<Note>
  The documentation will be available at `http://localhost:3000`
</Note>

### Workspace Settings

The repository includes predefined VSCode settings for optimal development:

- **Tab Size**: 2 spaces for all files
- **Word Wrap**: Enabled for better readability
- **Auto-formatting**: Disabled for markdown/MDX files
- **Trailing Whitespace**: Automatically trimmed
- **Final Newline**: Automatically inserted

## Writing Documentation

### File Format

All documentation files use MDX format, which combines Markdown with JSX components.

```mdx
---
title: 'Page Title'
description: 'Brief description of the page'
---

# Content goes here

<Card title="Example Card" icon="star">
  This is a Mintlify component
</Card>
```

### Available Components

Mintlify provides several built-in components:

<Tabs>
  <Tab title="Cards">
    ```mdx
    <Card title="Title" icon="icon-name" href="/link">
      Card content
    </Card>
    ```
  </Tab>
  <Tab title="Code Groups">
    Use CodeGroup for multiple code examples:
    ```javascript
    console.log('JavaScript example');
    ```
    ```python
    print('Python example')
    ```
  </Tab>
  <Tab title="Callouts">
    ```mdx
    <Note>This is a note</Note>
    <Warning>This is a warning</Warning>
    <Info>This is info</Info>
    <Tip>This is a tip</Tip>
    ```
  </Tab>
</Tabs>

## Deployment

The documentation is automatically deployed when changes are pushed to the main branch.

<Steps>
  <Step title="Create a feature branch">
    ```bash
    git checkout -b feature/your-feature-name
    ```
  </Step>
  <Step title="Make your changes">
    Edit the relevant MDX files and test locally
  </Step>
  <Step title="Commit your changes">
    ```bash
    git add .
    git commit -m "Add: your feature description"
    ```
  </Step>
  <Step title="Push to GitHub">
    ```bash
    git push origin feature/your-feature-name
    ```
  </Step>
  <Step title="Create a Pull Request">
    Open a PR on GitHub for review
  </Step>
</Steps>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Port 3000 is already in use">
    Kill the process using port 3000 or specify a different port:
    ```bash
    npm run dev -- --port 3001
    ```
  </Accordion>
  <Accordion title="Module not found errors">
    Clear the npm cache and reinstall:
    ```bash
    npm cache clean --force
    rm -rf node_modules package-lock.json
    npm install
    ```
  </Accordion>
  <Accordion title="MDX syntax errors">
    Ensure all JSX components are properly closed and markdown syntax is valid
  </Accordion>
</AccordionGroup>
